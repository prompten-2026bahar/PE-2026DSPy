# /translation/gepa_advanced.md

## dspy.GEPA - Advanced Features (Gelişmiş Özellikler)

### Özel Talimat Önericiler (Custom Instruction Proposers)

#### `instruction_proposer` nedir?
`instruction_proposer`, GEPA optimizasyonu sırasında `reflection_lm`'yi çağırmaktan ve yeni istemler (prompts) önermekten sorumlu bileşendir. GEPA, DSPy programınızda düşük performans gösteren bileşenleri belirlediğinde, talimat önerici (instruction proposer) yürütme izlerini (execution traces), geri bildirimleri ve hataları analiz ederek gözlemlenen sorunlara göre uyarlanmış iyileştirilmiş talimatlar üretir.

#### Varsayılan Uygulama (Default Implementation)
Varsayılan olarak GEPA, `ProposalFn` protokolünü uygulayan `GEPA` kütüphanesindeki yerleşik talimat önericiyi kullanır. **Varsayılan önerici** şu istem şablonunu (prompt template) kullanır:

```text
Bir asistana benim için bir görevi yerine getirmesi amacıyla aşağıdaki talimatları sağladım:
```<curr_instructions>```
Aşağıdakiler, asistana sağlanan farklı görev girdilerinin, asistanın her birine verdiği yanıtın ve asistanın yanıtının nasıl daha iyi olabileceğine dair bazı geri bildirimlerin örnekleridir:
```<inputs_outputs_feedback>```
Göreviniz asistan için yeni bir talimat yazmaktır.
Girdileri dikkatlice okuyun, girdi formatını tanımlayın ve asistanla çözmek istediğim görev hakkında detaylı görev açıklamasını çıkarın (infer).
Tüm asistan yanıtlarını ve ilgili geri bildirimleri okuyun. Görevle ilgili tüm niş ve alana özgü olgusal bilgileri belirleyin ve bunları talimata dahil edin, çünkü bunların birçoğu gelecekte asistan için mevcut olmayabilir. Asistan görevi çözmek için genellenebilir bir strateji kullanmış olabilir, eğer öyleyse, bunu da talimata dahil edin.
Yeni talimatları ``` blokları içinde sağlayın.
```

Bu şablon otomatik olarak şunlarla doldurulur:
* `<curr_instructions>`: Optimize edilmekte olan mevcut talimat.
* `<inputs_outputs_feedback>`: Tahminci girdilerini, üretilen çıktıları ve değerlendirme geri bildirimlerini içeren yapılandırılmış markdown metni.

Varsayılan davranış örneği:
```python
# Varsayılan talimat önerici otomatik olarak kullanılır
gepa = dspy.GEPA(
    metric=my_metric,
    reflection_lm=dspy.LM(model="gpt-5", temperature=1.0, max_tokens=32000, api_key=api_key),
    auto="medium"
)
optimized_program = gepa.compile(student, trainset=examples)
```

#### Özel `instruction_proposer` Ne Zaman Kullanılmalı?
**Not:** Özel talimat önericiler gelişmiş bir özelliktir. Çoğu kullanıcı, metin tabanlı optimizasyon görevlerinin çoğunda iyi çalışan varsayılan önericiyle başlamalıdır.

Şu durumlarda özel bir talimat önerici uygulamayı düşünebilirsiniz:
* **Çok modlu işleme (Multi-modal handling):** Girdilerinizdeki metinsel bilgilerin yanı sıra görüntüleri (`dspy.Image`) işlemek için.
* **Sınırlar ve uzunluk kısıtlamaları üzerinde nüanslı kontrol:** Talimat uzunluğu, formatı ve yapısal gereksinimler üzerinde daha ince taneli kontrole sahip olmak için.
* **Alana özgü bilgi (Domain-specific information):** Varsayılan önericinin sahip olmadığı ve `feedback_func` aracılığıyla sağlanamayan özel bilgileri, terminolojiyi veya bağlamı enjekte etmek için. (Bu gelişmiş bir özelliktir ve çoğu kullanıcının buna ihtiyacı olmamalıdır.)
* **Sağlayıcıya özgü istemleme kılavuzları:** Talimatları, kendilerine özgü formatlama tercihleri olan belirli LLM sağlayıcıları (OpenAI, Anthropic vb.) için optimize etmek.
* **Bağlantılı bileşen güncellemeleri (Coupled component updates):** Her bir bileşeni bağımsız olarak optimize etmek yerine, 2 veya daha fazla bileşenin koordineli bir şekilde birlikte güncellenmesi gereken durumları ele almak.
* **Harici bilgi entegrasyonu:** Talimat üretimi sırasında veritabanlarına, API'lere veya bilgi tabanlarına bağlanmak için.

#### Mevcut Seçenekler (Available Options)

**Yerleşik Seçenekler:**
* **Varsayılan Önerici (Default Proposer):** Standart GEPA talimat önericisi (`instruction_proposer=None` olduğunda kullanılır). Varsayılan talimat önericisi de BİR talimat önericidir! GEPA makalesi ve öğreticilerinde rapor edilen çeşitli deneyler için kullanılan en genel olanıdır.
* **MultiModalInstructionProposer:** `dspy.Image` girdilerini ve yapılandırılmış çok modlu içeriği işler.

```python
from dspy.teleprompt.gepa.instruction_proposal import MultiModalInstructionProposer

# Görüntüler veya çok modlu girdiler içeren görevler için
gepa = dspy.GEPA(
    metric=my_metric,
    reflection_lm=dspy.LM(model="gpt-5", temperature=1.0, max_tokens=32000, api_key=api_key),
    instruction_proposer=MultiModalInstructionProposer(),
    auto="medium"
)
```

GEPA kütüphanesi büyümeye devam ettikçe, uzmanlaşmış alanlar için yeni talimat önericilerine yönelik topluluk katkılarını davet ediyoruz.

#### Özel Talimat Önericiler Nasıl Uygulanır?
Özel talimat önericiler, çağrılabilir (callable) bir sınıf veya fonksiyon tanımlayarak `ProposalFn` protokolünü uygulamalıdır. GEPA, optimizasyon sırasında önericinizi çağıracaktır:

```python
from dspy.teleprompt.gepa.gepa_utils import ReflectiveExample

class CustomInstructionProposer:
    def __call__(
        self,
        candidate: dict[str, str],                          # Aday bileşen adı -> bu turda güncellenecek talimat eşlemesi
        reflective_dataset: dict[str, list[ReflectiveExample]],  # Bileşen -> şu yapıdaki örnekler: {"Inputs": ..., "Generated Outputs": ..., "Feedback": ...}
        components_to_update: list[str]                     # Hangi bileşenlerin iyileştirileceği
    ) -> dict[str, str]:                                    # Yalnızca güncellenmekte olan bileşenler için yeni talimat eşlemesini döndür
        # Özel talimat üretim mantığınız buraya
        return updated_instructions

# Veya bir fonksiyon olarak:
def custom_instruction_proposer(candidate, reflective_dataset, components_to_update):
    # Özel talimat üretim mantığınız buraya
    return updated_instructions
```

**Yansıtıcı Veri Seti Yapısı (Reflective Dataset Structure):**
* `dict[str, list[ReflectiveExample]]`: Bileşen adlarını örnek listelerine eşler.
* `ReflectiveExample` TypedDict şunları içerir:
  * `Inputs: dict[str, Any]`: Tahminci girdileri (`dspy.Image` nesnelerini içerebilir).
  * `Generated_Outputs: dict[str, Any] | str`: Başarı durumunda çıktı alanları sözlüğü, başarısızlık durumunda hata mesajı.
  * `Feedback: str`: Her zaman metrik fonksiyonundan gelen veya GEPA tarafından otomatik üretilen bir string'dir.

#### Temel Örnek: Kelime Sınırı Önericisi (Word Limit Proposer)
```python
import dspy
from gepa.core.adapter import ProposalFn
from dspy.teleprompt.gepa.gepa_utils import ReflectiveExample

class GenerateWordLimitedInstruction(dspy.Signature):
    '''Mevcut bir talimat ve geri bildirim örnekleri verildiğinde, kelime sınırı kısıtlamalarına sahip iyileştirilmiş bir talimat üretin.'''
    current_instruction = dspy.InputField(desc="İyileştirilmesi gereken mevcut talimat")
    feedback_summary = dspy.InputField(desc="Hem olumlu hem de olumsuz durumları içerebilecek örneklerden geri bildirim")
    max_words = dspy.InputField(desc="Yeni talimatta izin verilen maksimum kelime sayısı")
    improved_instruction = dspy.OutputField(desc="Max_words sınırının altında kalarak sorunları çözen yeni bir talimat")

class WordLimitProposer(ProposalFn):
    def __init__(self, max_words: int = 1000):
        self.max_words = max_words
        self.instruction_improver = dspy.ChainOfThought(GenerateWordLimitedInstruction)
        
    def __call__(self, candidate: dict[str, str], reflective_dataset: dict[str, list[ReflectiveExample]], components_to_update: list[str]) -> dict[str, str]:
        updated_components = {}
        for component_name in components_to_update:
            if component_name not in candidate or component_name not in reflective_dataset:
                continue
                
            current_instruction = candidate[component_name]
            component_examples = reflective_dataset[component_name]
            
            # Geri bildirim özeti oluştur
            feedback_text = "\n".join([
                f"Example {i+1}: {ex.get('Feedback', 'No feedback')}"
                for i, ex in enumerate(component_examples)  # Bağlam taşmasını önlemek için örnekleri sınırla
            ])
            
            # Talimatı iyileştirmek için modülü kullan
            result = self.instruction_improver(
                current_instruction=current_instruction,
                feedback_summary=feedback_text,
                max_words=self.max_words
            )
            updated_components[component_name] = result.improved_instruction
            
        return updated_components

# Kullanım
gepa = dspy.GEPA(
    metric=my_metric,
    reflection_lm=dspy.LM(model="gpt-5", temperature=1.0, max_tokens=32000, api_key=api_key),
    instruction_proposer=WordLimitProposer(max_words=700),
    auto="medium"
)
```

#### Gelişmiş Örnek: RAG Destekli Talimat Önericisi (RAG-Enhanced Instruction Proposer)
```python
import dspy
from gepa.core.adapter import ProposalFn
from dspy.teleprompt.gepa.gepa_utils import ReflectiveExample

class GenerateDocumentationQuery(dspy.Signature):
    '''Yaygın sorun desenlerini belirlemek ve ilgili dokümantasyonu almak üzere hedeflenmiş veritabanı sorguları üretmek için geri bildirimli örnekleri analiz edin.
    Amacınız, örneklerde bulunan problemli desenleri ele alan yönergeler için bir belge veritabanında arama yapmaktır.'''
    current_instruction = dspy.InputField(desc="İyileştirilmesi gereken mevcut talimat")
    examples_with_feedback = dspy.InputField(desc="Hangi sorunların oluştuğunu ve tekrarlayan desenleri gösteren geri bildirimli örnekler")
    failure_patterns: str = dspy.OutputField(desc="Örneklerde belirlenen yaygın başarısızlık desenlerini özetleyin")
    retrieval_queries: list[str] = dspy.OutputField(desc="Belirlenen problemli örnekleri ele alan ilgili dokümantasyonu bulmak için spesifik arama sorguları")

class GenerateRAGEnhancedInstruction(dspy.Signature):
    '''Alınan dokümantasyonu ve örnek analizini kullanarak iyileştirilmiş talimatlar üretin.'''
    current_instruction = dspy.InputField(desc="İyileştirilmesi gereken mevcut talimat")
    relevant_documentation = dspy.InputField(desc="Uzmanlaşmış dokümantasyondan alınan yönergeler ve en iyi uygulamalar")
    examples_with_feedback = dspy.InputField(desc="Mevcut talimatla ne gibi sorunlar oluştuğunu gösteren örnekler")
    improved_instruction: str = dspy.OutputField(desc="Alınan yönergeleri birleştiren ve örneklerde gösterilen sorunları ele alan geliştirilmiş talimat")

class RAGInstructionImprover(dspy.Module):
    '''Uzmanlaşmış dokümantasyonla talimatları iyileştirmek için RAG kullanan modül.'''
    def __init__(self, retrieval_model):
        super().__init__()
        self.retrieve = retrieval_model  # dspy.Retrieve veya özel bir retriever olabilir
        self.query_generator = dspy.ChainOfThought(GenerateDocumentationQuery)
        self.generate_answer = dspy.ChainOfThought(GenerateRAGEnhancedInstruction)

    def forward(self, current_instruction: str, component_examples: list):
        '''Alınan dokümantasyonu kullanarak talimatı iyileştirin.'''
        # LLM'nin örnekleri analiz etmesine ve hedeflenmiş geri getirme sorguları üretmesine izin verin
        query_result = self.query_generator(
            current_instruction=current_instruction,
            examples_with_feedback=component_examples
        )
        
        results = self.retrieve.query(
            query_texts=query_result.retrieval_queries,
            n_results=3
        )
        
        relevant_docs_parts = []
        for i, (query, query_docs) in enumerate(zip(query_result.retrieval_queries, results['documents'])):
            if query_docs:
                docs_formatted = "\n".join([f"  - {doc}" for doc in query_docs])
                relevant_docs_parts.append(
                    f"**Search Query #{i+1}**: {query}\n"
                    f"**Retrieved Guidelines**:\n{docs_formatted}"
                )
                
        relevant_docs = "\n\n" + "="*60 + "\n\n".join(relevant_docs_parts) + "\n" + "="*60
        
        # Alınan bağlamla iyileştirilmiş talimat üretin
        result = self.generate_answer(
            current_instruction=current_instruction,
            relevant_documentation=relevant_docs,
            examples_with_feedback=component_examples
        )
        return result

class DocumentationEnhancedProposer(ProposalFn):
    '''Uzmanlaşmış dokümantasyona RAG aracılığıyla erişen talimat önericisi.'''
    def __init__(self, documentation_retriever):
        self.instruction_improver = RAGInstructionImprover(documentation_retriever)

    def __call__(self, candidate: dict[str, str], reflective_dataset: dict[str, list[ReflectiveExample]], components_to_update: list[str]) -> dict[str, str]:
        updated_components = {}
        for component_name in components_to_update:
            if component_name not in candidate or component_name not in reflective_dataset:
                continue
                
            current_instruction = candidate[component_name]
            component_examples = reflective_dataset[component_name]
            
            result = self.instruction_improver(
                current_instruction=current_instruction,
                component_examples=component_examples
            )
            updated_components[component_name] = result.improved_instruction
            
        return updated_components

import chromadb
client = chromadb.Client()
collection = client.get_collection("instruction_guidelines")

gepa = dspy.GEPA(
    metric=task_specific_metric,
    reflection_lm=dspy.LM(model="gpt-5", temperature=1.0, max_tokens=32000, api_key=api_key),
    instruction_proposer=DocumentationEnhancedProposer(collection),
    auto="medium"
)
```

#### En İyi Uygulamalar (Best Practices):
* **DSPy'ın tam gücünü kullanın:** Doğrudan LM çağrıları yapmak yerine talimat önericinizi oluşturmak için `dspy.Module`, `dspy.Signature` ve `dspy.Predict` gibi DSPy bileşenlerinden yararlanın. Kısıtlamaları karşılamak için `dspy.Refine`, karmaşık akıl yürütme görevleri için `dspy.ChainOfThought` kullanmayı düşünün ve sofistike talimat iyileştirme iş akışları için birden fazla modülü birleştirin.
* **Bütünsel geri bildirim analizini etkinleştirin:** `dspy.GEPA`'nın `GEPAFeedbackMetric`'i aynı anda tek bir (altın/gold, tahmin/prediction) çiftini işlerken, talimat önericileri bir bileşen için "tüm örnekleri toplu olarak" (batch) alır; bu da örnekler arası örüntü tespitini ve sistematik sorun tanımlamasını mümkün kılar.
* **Veri serileştirmeye dikkat edin:** Her şeyi string'e dönüştürmek ideal olmayabilir; `dspy.Image` gibi karmaşık girdi türlerini daha iyi LM işlemesi için yapılarını koruyarak ele alın.
* **Kapsamlı test yapın:** Özel önericinizi temsili başarısızlık (failure) durumlarıyla test edin.

### Özel Bileşen Seçimi (Custom Component Selection)

#### `component_selector` nedir?
`component_selector` parametresi, her GEPA yinelemesinde (iteration) DSPy programınızdaki hangi bileşenlerin (tahmincilerin) optimizasyon için seçileceğini kontrol eder. Bir seferde bir bileşeni güncelleyen varsayılan "round-robin" yaklaşımı yerine, optimizasyon durumuna, performans yörüngelerine (trajectories) ve diğer bağlamsal bilgilere dayalı olarak tek veya birden fazla bileşeni seçen özel seçim stratejileri uygulayabilirsiniz.

#### Varsayılan Davranış (Default Behavior)
Varsayılan olarak GEPA, bileşenler arasında sırayla dönen ve her yinelemede bir bileşeni optimize eden bir **round-robin stratejisi** (`RoundRobinReflectionComponentSelector`) kullanır:

```python
# Varsayılan round-robin bileşen seçimi
gepa = dspy.GEPA(
    metric=my_metric,
    reflection_lm=dspy.LM(model="gpt-5", temperature=1.0, max_tokens=32000, api_key=api_key),
    # component_selector="round_robin"  # Bu varsayılandır
    auto="medium"
)
```

#### Yerleşik Seçim Stratejileri (Built-in Selection Strategies)
String tabanlı seçiciler:
* `"round_robin"` (varsayılan): Bileşenler arasında teker teker dolaşır.
* `"all"`: Eşzamanlı optimizasyon için tüm bileşenleri seçer.

```python
# Tüm bileşenleri eşzamanlı olarak optimize et
gepa = dspy.GEPA(
    metric=my_metric,
    reflection_lm=reflection_lm,
    component_selector="all",  # Tüm bileşenleri birlikte güncelle
    auto="medium"
)

# Açıkça round-robin seçimi
gepa = dspy.GEPA(
    metric=my_metric,
    reflection_lm=reflection_lm,
    component_selector="round_robin",  # Yineleme başına bir bileşen
    auto="medium"
)
```

#### Özel Bileşen Seçimi Ne Zaman Kullanılmalı?
Şu durumlarda özel bileşen seçimi uygulamayı düşünebilirsiniz:
* **Bağımlılığa duyarlı optimizasyon:** İlgili bileşenleri birlikte güncelleyin (örn. bir sınıflandırıcı ve onun girdi biçimlendiricisi).
* **LLM destekli seçim:** Bir LLM'nin yörüngeleri analiz etmesine ve hangi bileşenlerin dikkat gerektirdiğine karar vermesine izin verin.
* **Kaynağa duyarlı optimizasyon:** Optimizasyon kapsamlılığını hesaplama bütçesiyle dengeleyin.

#### Özel Bileşen Seçici Protokolü (Custom Component Selector Protocol)
Özel bileşen seçiciler, çağrılabilir bir sınıf veya fonksiyon tanımlayarak `ReflectionComponentSelector` protokolünü uygulamalıdır. GEPA, optimizasyon sırasında seçicinizi çağıracaktır:

```python
from dspy.teleprompt.gepa.gepa_utils import GEPAState, Trajectory

class CustomComponentSelector:
    def __call__(
        self,
        state: GEPAState,                    # Geçmişi de içeren tam optimizasyon durumu
        trajectories: list[Trajectory],      # Mevcut mini yığından gelen yürütme izleri
        subsample_scores: list[float],       # Mevcut mini yığındaki her örnek için skorlar
        candidate_idx: int,                  # Optimize edilmekte olan mevcut program adayının indeksi
        candidate: dict[str, str],           # Bileşen adı -> talimat eşlemesi
    ) -> list[str]:                          # Optimize edilecek bileşen adlarının listesini döndür
        # Özel bileşen seçimi mantığınız buraya
        return selected_components

# Veya bir fonksiyon olarak:
def custom_component_selector(state, trajectories, subsample_scores, candidate_idx, candidate):
    # Özel bileşen seçimi mantığınız buraya
    return selected_components
```

#### Özel Uygulama Örneği (Custom Implementation Example)
İşte bileşenlerinizin farklı yarılarını optimize etmek arasında geçiş yapan basit bir fonksiyon:

```python
def alternating_half_selector(state, trajectories, subsample_scores, candidate_idx, candidate):
    '''Çift yinelemelerde bileşenlerin yarısını, tek yinelemelerde diğer yarısını optimize edin.'''
    components = list(candidate.keys())
    
    # Yalnızca bir bileşen varsa, onu her zaman optimize edin
    if len(components) <= 1:
        return components
        
    mid_point = len(components) // 2
    
    # Yarılar arasında geçiş yapmak için state.i (yineleme sayacı) kullanın
    if state.i % 2 == 0:
        # Çift yineleme: ilk yarıyı optimize et
        return components[:mid_point]
    else:
        # Tek yineleme: ikinci yarıyı optimize et
        return components[mid_point:]

# Kullanım
gepa = dspy.GEPA(
    metric=my_metric,
    reflection_lm=reflection_lm,
    component_selector=alternating_half_selector,
    auto="medium"
)
```

#### Özel Talimat Önericilerle Entegrasyon (Integration with Custom Instruction Proposers)
Bileşen seçiciler, özel talimat önericilerle sorunsuz bir şekilde çalışır. Seçici hangi bileşenlerin güncelleneceğini belirler, ardından talimat önerici o bileşenler için yeni talimatlar üretir:

```python
# Birleştirilmiş özel seçici + özel önerici
gepa = dspy.GEPA(
    metric=my_metric,
    reflection_lm=reflection_lm,
    component_selector=alternating_half_selector,
    instruction_proposer=WordLimitProposer(max_words=500),
    auto="medium"
)
```
