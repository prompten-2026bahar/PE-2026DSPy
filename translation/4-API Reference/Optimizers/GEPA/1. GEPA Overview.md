## dspy.GEPA: Reflective Prompt Optimizer

**GEPA (Genetic-Pareto)**, "GEPA: Reflective Prompt Evolution Can Outperform Reinforcement Learning" (Agrawal ve ark., 2025, arxiv:2507.19457) makalesinde önerilen, isteğe bağlı sistemlerin metinsel bileşenlerini (istemler/prompts gibi) adaptif olarak evrimleştiren yansıtıcı (reflective) bir optimize edicidir. Metrikler tarafından döndürülen skaler puanlara ek olarak, kullanıcılar optimizasyon sürecini yönlendirmek için GEPA'ya metin tabanlı bir geri bildirim de sağlayabilirler. Bu tür metinsel geri bildirimler, GEPA'ya sistemin neden o puanı aldığına dair daha fazla görünürlük sağlar ve ardından GEPA skoru nasıl iyileştirebileceğini belirlemek için kendi içine bakabilir (introspect). Bu, GEPA'nın çok az sayıda döngüde (rollout) yüksek performanslı istemler önermesine olanak tanır.

```python
dspy.GEPA(metric: GEPAFeedbackMetric, *, auto: Literal['light', 'medium', 'heavy'] | None = None, max_full_evals: int | None = None, max_metric_calls: int | None = None, reflection_minibatch_size: int = 3, candidate_selection_strategy: Literal['pareto', 'current_best'] = 'pareto', reflection_lm: LM | None = None, skip_perfect_score: bool = True, add_format_failure_as_feedback: bool = False, instruction_proposer: ProposalFn | None = None, component_selector: ReflectionComponentSelector | str = 'round_robin', use_merge: bool = True, max_merge_invocations: int | None = 5, num_threads: int | None = None, failure_score: float = 0.0, perfect_score: float = 1.0, log_dir: str | None = None, track_stats: bool = False, use_wandb: bool = False, wandb_api_key: str | None = None, wandb_init_kwargs: dict[str, Any] | None = None, track_best_outputs: bool = False, warn_on_score_mismatch: bool = True, use_mlflow: bool = False, seed: int | None = 0, gepa_kwargs: dict | None = None)
```

**Kullanılan Yapılar (Bases):** `Teleprompter`

GEPA, karmaşık sistemlerin metin bileşenlerini evrimleştirmek için yansıma (reflection) kullanan evrimsel bir optimize edicidir. GEPA, "GEPA: Reflective Prompt Evolution Can Outperform Reinforcement Learning" adlı makalede önerilmiştir. GEPA optimizasyon motoru, `https://github.com/gepa-ai/gepa` adresinde bulunan `gepa` paketi tarafından sağlanır.

GEPA, DSPy modülünün yürütülmesinin (execution) tam izlerini (traces) yakalar, izlemenin belirli bir tahminciye (predictor) karşılık gelen kısımlarını tanımlar ve tahminci için yeni bir talimat (instruction) önermek üzere tahmincinin davranışı üzerine yansıma yapar (reflects). GEPA, kullanıcıların tahmincinin evrimini yönlendirmek için kullanılan metinsel geri bildirimleri optimize ediciye sağlamasına olanak tanır. Metinsel geri bildirim, bireysel tahminciler düzeyinde veya tüm sistemin yürütülmesi düzeyinde sağlanabilir.

GEPA optimize edicisine geri bildirim sağlamak için aşağıdaki gibi bir metrik uygulayın:

```python
def metric(
    gold: Example,
    pred: Prediction,
    trace: Optional[DSPyTrace] = None,
    pred_name: Optional[str] = None,
    pred_trace: Optional[DSPyTrace] = None,
) -> float | ScoreWithFeedback:
    '''
    Bu fonksiyon aşağıdaki argümanlarla çağrılır:
    - gold: Altın (doğru) örnek.
    - pred: Tahmin edilen çıktı.
    - trace: İsteğe bağlı. Program yürütmesinin izi (trace).
    - pred_name: İsteğe bağlı. GEPA tarafından o anda optimize edilen ve geri bildirim istenen hedef tahmincinin adı.
    - pred_trace: İsteğe bağlı. GEPA'nın geri bildirim aradığı hedef tahmincinin yürütülmesinin izi.

    `pred_name` ve `pred_trace` argümanlarına dikkat edin. Optimizasyon sırasında GEPA, optimize edilen bireysel tahminciler için geri bildirim almak üzere metriği çağıracaktır. GEPA, tahmincinin adını `pred_name` içinde ve izlemenin tahminciye karşılık gelen alt izlemesini `pred_trace` içinde sağlar.
    
    Tahminci düzeyinde mevcutsa, metrik tahminciye karşılık gelen {'score': float, 'feedback': str} döndürmelidir.
    Tahminci düzeyinde mevcut değilse, metrik program düzeyinde de (yalnızca gold, pred ve trace kullanarak) bir metin geri bildirimi döndürebilir.
    Hiçbir geri bildirim döndürülmezse GEPA, yalnızca skordan oluşan basit bir metin geri bildirimi kullanacaktır:
    f"This trajectory got a score of {score}."
    '''
    ...
```

GEPA ayrıca, `valset=trainset, track_stats=True, track_best_outputs=True` parametreleri geçilerek ve yığının Pareto cephesini (Pareto frontier) elde etmek için optimize edilmiş programın (`compile` tarafından döndürülen) `detailed_results` özniteliği kullanılarak bir toplu çıkarım zamanı (batch inference-time) arama stratejisi olarak da kullanılabilir. `optimized_program.detailed_results.best_outputs_valset`, yığındaki her bir görev için en iyi çıktıları içerecektir.

**Örnekler (Examples):**

```python
gepa = GEPA(metric=metric, track_stats=True)
batch_of_tasks = [dspy.Example(...) for task in tasks]
new_prog = gepa.compile(student, trainset=trainset, valset=batch_of_tasks)
pareto_frontier = new_prog.detailed_results.val_aggregate_scores
# pareto_frontier, yığındaki her görev için bir tane olmak üzere skorların bir listesidir.
```

* **Yansıma Yapılandırması (Reflection Configuration):** `reflection_lm` parametresi gereklidir ve güçlü bir dil modeli olmalıdır. Yansıma süreci, başarısız örnekleri analiz ederek programı iyileştirmek için geri bildirim üretir.

* **Değerlendirme Yapılandırması (Evaluation Configuration):** Değerlendirmeyi paralelleştirmek için `num_threads` kullanın.
* **Loglama Yapılandırması (Logging Configuration):** Detaylı günlükleri kaydetmek ve kontrol noktasından (checkpoint) devam etmeyi etkinleştirmek için `log_dir` ayarlayın. Optimize edilen sonuçlara erişmek için `track_stats=True` kullanın.
* **Tekrarlanabilirlik (Reproducibility):** Tutarlı sonuçlar için `seed` ayarlayın.

## dspy.teleprompt.gepa.gepa.GEPAFeedbackMetric

GEPA'nın arkasındaki temel içgörülerden biri, alana özgü (domain-specific) metinsel geri bildirimlerden yararlanma yeteneğidir. Kullanıcılar GEPA metriği olarak aşağıdaki çağrı imzasına (call signature) sahip bir geri bildirim fonksiyonu sağlamalıdır:

### Fonksiyonlar (Functions)

#### `__call__`

```python
__call__(gold: Example, pred: Prediction, trace: Optional[DSPyTrace], pred_name: str | None, pred_trace: Optional[DSPyTrace]) -> Union[float, ScoreWithFeedback]
```

Bu fonksiyon aşağıdaki argümanlarla çağrılır:
- `gold`: Altın (doğru) örnek.
- `pred`: Tahmin edilen çıktı.
- `trace`: İsteğe bağlı. Programın yürütülmesinin izi.
- `pred_name`: İsteğe bağlı. GEPA tarafından o anda optimize edilen ve geri bildirim istenen hedef tahmincinin adı.
- `pred_trace`: İsteğe bağlı. GEPA'nın geri bildirim aradığı hedef tahmincinin yürütülmesinin izi.

`pred_name` ve `pred_trace` argümanlarına dikkat edin. Optimizasyon sırasında GEPA, optimize edilen bireysel tahminciler için geri bildirim almak üzere metriği çağıracaktır. Tahminci düzeyinde mevcutsa, metrik `dspy.Prediction(score: float, feedback: str)` döndürmelidir. Tahminci düzeyinde mevcut değilse, metrik program düzeyinde de (yalnızca gold, pred ve trace kullanarak) bir metin geri bildirimi döndürebilir.

## dspy.teleprompt.gepa.gepa.DspyGEPAResult

`track_stats=True` olduğunda, GEPA önerilen tüm adaylar hakkında detaylı sonuçlar ve optimizasyon çalışması hakkında meta veriler döndürür. Sonuçlar, GEPA tarafından döndürülen optimize edilmiş programın `detailed_results` özniteliğinde mevcuttur ve aşağıdaki veri sınıfı (dataclass) tipine sahiptir:

```python
dspy.teleprompt.gepa.gepa.DspyGEPAResult(candidates: list[Module], parents: list[list[int | None]], val_aggregate_scores: list[float], val_subscores: list[list[float]], per_val_instance_best_candidates: list[set[int]], discovery_eval_counts: list[int], best_outputs_valset: list[list[tuple[int, list[Prediction]]]] | None = None, total_metric_calls: int | None = None, num_full_val_evals: int | None = None, log_dir: str | None = None, seed: int | None = None) # dataclass
```

GEPA çalıştırmasıyla ilgili ek veriler.

**Alanlar (Fields):**
- `candidates`: Önerilen adayların listesi (component_name -> component_text)
- `parents`: Soy bilgisi (lineage info); her bir $i$ adayı için `parents[i]`, ebeveyn indekslerinin veya None değerlerinin bir listesidir.
- `val_aggregate_scores`: Doğrulama seti üzerinde aday başına elde edilen toplam skor (yüksek olması daha iyidir).
- `val_subscores`: Doğrulama seti üzerinde aday başına ve örnek başına (per-instance) skorlar (uzunluk == num_val_instances).
- `per_val_instance_best_candidates`: Her bir doğrulama örneği $t$ için, $t$ üzerinde en iyi skoru elde eden aday indekslerinin bir kümesi.
- `discovery_eval_counts`: Her bir adayın keşfedilmesine kadar tüketilen bütçe (metrik çağrısı / rollout sayısı).
- `total_metric_calls`: Çalıştırma boyunca yapılan toplam metrik çağrısı sayısı.
- `num_full_val_evals`: Gerçekleştirilen tam doğrulama değerlendirmesi sayısı.
- `log_dir`: Eserlerin/logların yazıldığı dizin (eğer varsa).
- `seed`: Tekrarlanabilirlik için RNG tohumu (biliniyorsa).
- `best_idx`: En yüksek `val_aggregate_scores` değerine sahip adayın indeksi.
- `best_candidate`: `best_idx` için program metni eşlemesi.

### Çıkarım-Zamanı Araması (Inference-Time Search)

GEPA, bir test zamanı/çıkarım (inference) arama mekanizması olarak görev yapabilir. `valset` değerinizi değerlendirme yığını (evaluation batch) olarak ayarlayıp `track_best_outputs=True` kullanarak, GEPA evrimsel arama sırasında her bir yığın elemanı için bulunan en yüksek skorlu çıktıları üretir.

```python
gepa = dspy.GEPA(metric=metric, track_stats=True, ...)
new_prog = gepa.compile(student, trainset=my_tasks, valset=my_tasks)

highest_score_achieved_per_task = new_prog.detailed_results.highest_score_achieved_per_val_task
best_outputs = new_prog.detailed_results.best_outputs_valset
```

## GEPA Nasıl Çalışır? (How Does GEPA Work?)

**1. Yansıtıcı İstem Mutasyonu (Reflective Prompt Mutation)**
GEPA, yapılandırılmış yürütme izleri (girdiler, çıktılar, hatalar, geri bildirimler) üzerine yansıma (reflect) yapmak için YZ Modellerini (LLM) kullanır, seçilen bir modülü hedefler ve gerçek gözlemlenen hatalara ve zengin metinsel/çevresel geri bildirimlere göre uyarlanmış yeni bir talimat/program metni önerir.

**2. Optimizasyon Sinyali Olarak Zengin Metinsel Geri Bildirim**
GEPA, yalnızca skaler ödülleri değil, mevcut olan "herhangi bir" metinsel geri bildirimi kullanabilir. Buna değerlendirme günlükleri, kod izleri, ayrıştırma hataları, kısıtlama ihlalleri, hata mesajı dizeleri veya hatta izole edilmiş alt modüle özgü geri bildirimler dahildir. Bu, eyleme geçirilebilir ve alanın farkında olan (domain-aware) bir optimizasyona izin verir.

**3. Pareto Tabanlı Aday Seçimi (Pareto-based Candidate Selection)**
Yalnızca "en iyi" küresel adayı (global candidate) evrimleştirmek (ki bu yerel optimumlara veya durgunluğa yol açar) yerine GEPA bir Pareto cephesi sürdürür: en az bir değerlendirme örneğinde en yüksek puanı elde eden adayların kümesi. Her yinelemede mutasyona uğrayacak bir sonraki aday bu cepheden (kapsamla orantılı bir olasılıkla) örneklenir, böylece hem keşif (exploration) hem de birbirini tamamlayan stratejilerin sağlam bir şekilde korunması garanti altına alınır.

### Algoritma Özeti (Algorithm Summary)

1. Aday havuzunu optimize edilmemiş programla **başlatın** (Initialize).
2. **Yineleyin (Iterate):**
    * (Pareto cephesinden) Bir aday **örnekleyin**.
    * Eğitim setinden bir mini yığın **örnekleyin**.
    * Mini yığın üzerindeki modül yürütmesi için yürütme izlerini + geri bildirimleri **toplayın**.
    * Hedeflenen iyileştirme için adayın bir modülünü **seçin**.
    * **LLM Yansıması (Reflection):** Yansıtıcı meta-istemleme ve toplanan geri bildirimi kullanarak hedeflenen modül için yeni bir talimat/istem önerin.
    * Yeni adayı mini yığın üzerinde **çalıştırın**; iyileşme varsa, Pareto doğrulama seti üzerinde değerlendirin.
    * Aday havuzunu / Pareto cephesini **güncelleyin**.
    * *[İsteğe bağlı]* Sistemin farkında olan birleştirme/çaprazlama: Farklı soy hatlarından en iyi performans gösteren modülleri birleştirin.
3. Yürütme (rollout) veya metrik bütçesi tükenene kadar **devam edin**.
4. Doğrulama üzerinde en iyi toplam performansa sahip adayı **döndürün**.

## Geri Bildirim Metriklerinin Uygulanması (Implementing Feedback Metrics)

İyi tasarlanmış bir metrik, GEPA'nın örnek verimliliğinin (sample efficiency) ve öğrenme sinyali zenginliğinin merkezinde yer alır. GEPA, metriğin bir `dspy.Prediction(score=..., feedback=...)` döndürmesini bekler. GEPA, LLM tabanlı iş akışlarından gelen doğal dil izlerini optimizasyon için kullanır; ara yörüngeleri ve hataları sayısal ödüllere indirgemek yerine düz metin olarak korur. Bu, insan teşhis süreçlerini yansıtarak sistem davranışlarının ve darboğazların daha net tanımlanmasını sağlar.

**GEPA Dostu Geri Bildirim İçin Pratik Reçete:**
* **Mevcut Eserlerden Yararlanın:** Logları, birim testlerini (unit tests), değerlendirme komut dosyalarını ve profil oluşturucu (profiler) çıktılarını kullanın; bunları yüzeye çıkarmak genellikle yeterlidir.
* **Sonuçları Ayrıştırın (Decompose):** Skorları hedef başına bileşenlere (örn. doğruluk, gecikme, maliyet, güvenlik) ayırın ve hataları adımlara bağlayın.
* **Yörüngeleri Ortaya Çıkarın:** Boru hattı aşamalarını etiketleyin, belirgin hatalarla (örn. kod oluşturma boru hatlarında) geçiş/başarısızlık durumlarını raporlayın.
* **Kontrollere Dayandırın:** Doğrulanamayan görevler için otomatik doğrulayıcılar (birim testleri, şemalar, simülatörler) veya LLM-as-a-judge (yargıç LLM) kullanın (PUPA'da olduğu gibi).
* **Netliğe Öncelik Verin:** Teknik karmaşıklıktan ziyade hata kapsamına ve karar noktalarına odaklanın.

**Örnekler (Examples):**
* **Belge Getirme (örn. HotpotQA):** Sadece Recall/F1 skorlarının ötesinde doğru getirilen, yanlış olan veya kaçırılan belgeleri listeleyin.
* **Çok Hedefli Görevler (örn. PUPA):** Her hedefin katkılarını ortaya çıkarmak için toplam skorları ayrıştırın, ödünleşimleri (tradeoffs) vurgulayın (örn. kaliteye karşı gizlilik).
* **Yığılmış Boru Hatları (örn. kod oluşturma: ayrıştır → derle → çalıştır → profille → değerlendir):** Aşamaya özgü hataları ortaya çıkarın; doğal dil izleri genellikle LLM'nin kendi kendini düzeltmesi için yeterlidir.
